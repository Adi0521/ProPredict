# 4.5 ‚Äî MD Simulation Validation

**Date:** 2026-06-24
**Status:** Complete ‚Äî 30 tests passing in the `ProPredict` conda env

## What was done

Added a post-simulation sanity check that escalates a prediction when the MD
trajectory looks unphysical (blown-up run). When validation fails, the pipeline
sets `post_proc.decision = "escalate"` and records why.

### `validate_simulation_metrics(sim_result) -> Optional[str]`  (`orchestrator/scoring.py`)

Returns `None` when the trajectory looks sane, else a human-readable reason. Checks:

1. **Potential energy finite** ‚Äî fails on NaN/inf.
2. **RMSD blowup** ‚Äî fails if `max(rmsd_nm) > SIM_RMSD_MAX_NM` (2.0 nm). Falls back
   to the `rmsd_final_nm` scalar when no per-frame list is present (GROMACS).
3. **Rg divergence** ‚Äî fails if `max(rg_nm) > SIM_RG_DIVERGENCE_FACTOR x rg_nm[0]`
   (3x the initial frame).

Missing/empty metrics are treated as "can't check" (returns `None`) so a partial
result never produces a false escalation.

### Wiring (`orchestrator/tasks.py`, MD stage ~line 367)

Runs immediately after `post_proc.simulation_metrics = sim_result`, inside the
existing `try`. The MD stage is terminal (only result-dict assembly follows) and
runs *after* the refinement loop, so overriding `decision` here is safe and does
not interfere with refinement.

```python
failure_reason = validate_simulation_metrics(sim_result)
if failure_reason:
    logger.warning(f"MD validation failed ‚Äî escalating: {failure_reason}")
    post_proc.validation_reason = failure_reason
    post_proc.decision = "escalate"
```

## Decisions (and deviations from nextsteps.md)

- **Location: `scoring.py`, not `tasks.py`** (plan said tasks.py). It's a pure
  function, sits next to `compute_post_processing`, and is unit-testable without
  pulling in Celery.
- **Signature returns `Optional[str]` reason, not `-> bool`** (plan said bool). The
  plan also said "escalate *with a reason*" ‚Äî returning the reason is what lets us
  surface it. `None` = valid.
- **New schema field `validation_reason: Optional[str]`** on `PostProcessingResult`
  rather than overloading `agent_reasoning` (which is the Stage D Claude agent's
  explanation). Keeps the two sources attributable. Optional, defaults to `None`,
  so nothing constructing the model breaks.
- **Two new config flags** `SIM_RMSD_MAX_NM=2.0`, `SIM_RG_DIVERGENCE_FACTOR=3.0`
  in `config.py` + `.env.example` (NaN needs no threshold).
- **Tolerant of both backends' shapes** ‚Äî OpenMM always returns `rmsd_nm`/`rg_nm`
  lists; GROMACS may return only `rmsd_final_nm`. Handled both.
