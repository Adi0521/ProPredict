# 4.1 — Split requirements.txt

**Date:** 2026-06-24
**Status:** Complete

## What changed

Split the monolithic `requirements.txt` into three files:

- **`requirements.txt`** — core pip-installable deps. Cleaned up by removing the commented-out Boltz install instructions and the conda-managed comments block. All 22 packages here install cleanly via `pip install -r requirements.txt`.
- **`requirements-gpu.txt`** — extends base via `-r requirements.txt`, adds `boltz` from source. Use when `BOLTZ_ENABLED=True` and a GPU is available.
- **`environment-conda.yml`** — conda environment file for deps that lack PyPI wheels on ARM64 or are conda-only: openmm, pdbfixer, rdkit, openff-toolkit, vina, pyrosetta, mdanalysis.

## Decisions

- **meeko stays in `requirements.txt`** — it's on PyPI and pip-installs cleanly. The previous audit in nextsteps.md had it listed as pip-unfriendly, but that was incorrect (see: https://pypi.org/project/meeko/).
- **GROMACS is documented as a comment** in the conda file rather than a dependency — it's installed via brew (macOS) or apt-get (Docker), not conda.
- **pyrosetta included in conda file** — it requires the rosettacommons channel and a free academic license.
- Used `environment-conda.yml` (not `requirements-conda.txt`) because conda's YAML format is the standard way to specify channels and cross-platform deps.
