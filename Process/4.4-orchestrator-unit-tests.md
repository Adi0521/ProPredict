# 4.4 — Unit tests for core orchestrator functions

**Date:** 2026-06-24
**Status:** Complete — 22 tests passing in the `ProPredict` conda env

## What was done

Added `tests/test_orchestrator.py` covering the five core helper functions named
in nextsteps.md 4.4. The functions had been refactored out of `tasks.py` into
separate modules since the plan was written, so the tests target their real
locations:

| Function | Real location | Coverage |
|----------|---------------|----------|
| `_parse_plddt_from_pdb` | `orchestrator/backends/esmfold.py` | B-factor 0–1 → 0–100 scaling; non-CA / malformed lines ignored; empty input |
| `count_clashes` | `orchestrator/scoring.py` | non-adjacent clash detected; adjacent (\|i−j\|≤1) excluded; far-apart → 0 |
| `generate_cache_key` | `orchestrator/tasks.py` | determinism; context key-order invariance; sequence/context/pipeline all change the key |
| `compute_post_processing` | `orchestrator/scoring.py` | accept/refine/escalate thresholds incl. boundaries (75, 60); clash score penalty |
| `_determine_protonation_states` | `orchestrator/simulation.py` | HIS/ASP/GLU codes at pH 7.4 and pH 2.0; custom `pka_dict` override |

## Decisions

- **Shared `_atom_line()` helper** emits column-exact PDB `ATOM` records so the
  manual slicing in `_parse_plddt_from_pdb` (name `[12:16]`, B-factor `[60:66]`)
  and `_get_titratable_residues` (resName `[17:20]`, chain `[21]`, resSeq
  `[22:26]`) all read the right fields, and BioPython's `PDBParser` parses them too.
- **Thresholds and pKa values are read from the code, not assumed** — accept ≥75 /
  refine ≥60 from `config.py`; model pKa HIS 6.0 / ASP 3.9 / GLU 4.1 from
  `simulation._MODEL_PKA`. Tests assert real behavior.
- **`compute_post_processing` patches `orchestrator.scoring.count_clashes` → 0**
  so the threshold tests are isolated from BioPython geometry. `count_clashes`
  itself is covered directly in its own class.
- **BioPython scoped to its own class** via `@pytest.mark.skipif(not _has_biopython())`
  rather than a module-level `importorskip`, so the other 4 functions' tests still
  run even if BioPython is missing. (An earlier module-level `importorskip` wrongly
  skipped the whole file.)
